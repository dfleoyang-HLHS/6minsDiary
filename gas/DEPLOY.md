# Apps Script 部署指南

## 錯誤：No HTML file named index was found

這代表 Apps Script 專案裡**缺少名為 `index` 的 HTML 檔案**。

左側檔案清單必須同時有這兩個檔案：

```
📁 專案
 ├── Code.gs        ← 後端程式
 └── index.html     ← 前端網頁（名稱必須是 index）
```

---

## 正確建立步驟（請逐步照做）

### 步驟 1：建立 Code.gs

1. 開啟 [script.google.com](https://script.google.com/) → **新增專案**
2. 刪除編輯器裡預設的 `function myFunction() {...}`
3. 將本 repo 的 `gas/Code.gs` **完整內容**貼上
4. 確認檔名為 `Code.gs`（預設即是）

**（建議）啟用 manifest 檔案以設定權限：**

1. 點左側 **專案設定**（齒輪圖示）
2. 勾選 **在編輯器中顯示 appsscript.json 資訊清單檔案**
3. 左側會出現 `appsscript.json`，貼上 repo 中 `gas/appsscript.json` 的內容（含 `oauthScopes` 雲端硬碟權限）

### 步驟 2：建立 index HTML 檔（最容易漏掉）

1. 點左側 **+**（新增檔案）
2. 選 **HTML**
3. 檔名輸入 **`index`**（全小寫，不要加 .html，不要取名 Index 或其他名稱）
4. 刪除編輯器裡預設的 `<!DOCTYPE html>...` 內容
5. 將本 repo 的 `gas/index.html` **完整內容**貼上
6. 按 **Ctrl+S**（或 Cmd+S）儲存

### 步驟 3：確認檔案清單

左側應顯示：

| 檔案 | 類型 |
|------|------|
| `Code.gs` | 指令碼 |
| `index` | HTML |

若只有 `Code.gs` 沒有 `index`，就會出現你遇到的錯誤。

### 步驟 4：部署前測試（這不是部署！）

1. 在 `Code.gs` 上方函式選單選 **`testDeployment`**
2. 點 **執行**（▶ 播放按鈕）
3. 第一次會要求 **授權** → 點「檢閱權限」→ 選你的帳號 → 「進階」→ 「前往...（不安全）」→ **允許**
4. 執行日誌若顯示 `✅ index 檔案存在，可以部署` 即代表檔案設定正確

> ⚠️ **「執行」≠「部署」**  
> 日誌只顯示 `Execution started` / `Execution completed` 代表**測試跑完**，不代表網頁已上線。  
> 還必須完成下方步驟 5 的「部署」才會有網址。

### 步驟 5：部署（取得網址的關鍵步驟）

1. 點右上角 **部署** → **新增部署作業**（不是「測試部署」）
2. 點齒輪圖示 ⚙ → 類型選 **網頁應用程式**
3. 設定：

| 項目 | 值 |
|------|-----|
| 說明 | 任意，如「v1」 |
| 執行身分 | **存取網頁應用程式的使用者** |
| 誰可以存取 | **所有人**（若失敗可改「所有已登入 Google 帳戶的使用者」） |

4. 點 **部署**
5. **第二次授權**（部署時會再問一次）：
   - 若出現「Google 尚未驗證這個應用程式」
   - 點 **進階** → **前往 6分鐘魔法日記本（不安全）** → **允許**
6. 成功後會跳出視窗，顯示 **網頁應用程式 URL**（`https://script.google.com/macros/s/.../exec`）
7. 複製此 URL，用瀏覽器開啟測試

**部署成功的標誌：** 有 `script.google.com/macros/s/.../exec` 網址，且開啟後能看到日記頁面。

## 執行成功但部署失敗？對照檢查

| 現象 | 原因 | 解法 |
|------|------|------|
| 日誌只有 Execution started/completed | 只按了「執行」，沒做「部署」 | 依步驟 5 做 **新增部署作業** |
| 按部署後沒反應 | 授權視窗被瀏覽器擋住 | 允許彈出視窗，或看網址列是否有封鎖圖示 |
| 出現「需要授權」但沒完成 | 關閉了授權視窗 | 重新部署，完整走完「檢閱權限 → 允許」 |
| 「Google 尚未驗證這個應用程式」 | 個人開發的未驗證 App 正常現象 | 點 **進階** → **前往...（不安全）** → 允許 |
| 部署按鈕灰色或無法點 | 檔案未儲存 | Ctrl+S 儲存後再試 |
| 選錯部署類型 | 選成 API 可執行檔等 | 必須選 **網頁應用程式** |
| 公司/學校 Gmail | 管理員限制外部 App | 改用個人 Gmail，或請管理員開放 |
| 部署完但開網址是空白 | 用了舊版本 | 管理部署作業 → 編輯 → 版本選 **新版本** → 部署 |

## 授權流程圖（部署時必經）

```
按「部署」
    ↓
出現「需要授權」→ 點「授權存取權」
    ↓
選擇你的 Google 帳號
    ↓
「Google 尚未驗證」→ 點「進階」→「前往...（不安全）」
    ↓
勾選權限說明 → 點「允許」
    ↓
出現「網頁應用程式 URL」→ 部署成功 ✅
```

若在任何一步按了「取消」或關閉視窗，部署就不會完成。

## 常見錯誤

| 錯誤 | 原因 | 解法 |
|------|------|------|
| No HTML file named index | 沒建立 index 檔 | 依步驟 2 新增 HTML 檔，命名 `index` |
| No HTML file named styles | 舊版需要 styles 檔 | 請更新為最新版，只需 `index` 一個 HTML 檔 |
| 檔名是 Index（大寫 I） | 大小寫不符 | 刪除後重建，命名 `index` |
| 改了程式但網頁沒變 | 未重新部署 | 部署 → **管理部署作業** → 編輯 → **版本：新版本** → 部署 |

## 更新程式後

每次修改 `Code.gs` 或 `index` 後：

1. **儲存**所有檔案
2. **部署** → **管理部署作業**
3. 點鉛筆圖示（編輯）
4. **版本** 選 **新版本**
5. 點 **部署**

否則網址雖然能開，但會跑舊版程式。

## 錯誤：You do not have permission to call DriveApp

完整訊息類似：
`You do not have permission to call DriveApp.getFoldersByName`

### ⚠️ 若錯誤包含 `getFoldersByName`，代表仍在跑舊版程式！

**請在 Apps Script 的 Code.gs 按 Ctrl+F 搜尋 `getFoldersByName`：**

| 搜尋結果 | 代表 |
|----------|------|
| **有找到** | 仍是舊版，必須整份替換 Code.gs |
| **找不到** | 已是新版，請重新部署並清除舊授權 |

### 強制更新步驟（請逐步執行）

1. 開啟 Apps Script 的 `Code.gs`
2. **全選刪除**（Ctrl+A → Delete）
3. 從本 repo 複製 **整份** `gas/Code.gs` 貼上
4. 確認第 6 行有 `var SCRIPT_VERSION = "2.1";`
5. 確認搜尋 `getFoldersByName` → **0 筆結果**
6. **儲存**（Ctrl+S）
7. 執行 `testDeployment` → 授權 → 日誌應顯示 `版本：2.1`
8. **部署** → **管理部署作業** → 編輯 → **版本：新版本** → **部署**
9. [移除舊授權](https://myaccount.google.com/permissions) → 重新開啟網址 → 再授權

### 成功標誌

- 網頁頁尾顯示 **版本 2.1**
- 不再出現 `getFoldersByName` 錯誤
- 可正常寫入日記

### 原因（技術說明）

舊版用 `DriveApp.getFoldersByName()` 搜尋整個雲端硬碟，需要完整 drive 權限。
新版 v2.1 改為 `DriveApp.createFolder()` + 記錄資料夾 ID，只需 drive.file 權限。

若只更新了 `appsscript.json` 但 Code.gs 仍是舊版，授權後仍會失敗。

### 其他原因

#### A. 開發者（你）在 Apps Script 編輯器

1. 將最新的 `Code.gs` 和 `appsscript.json` 內容貼上（新版已修正權限問題）
2. **儲存**所有檔案
3. 函式選單選 **`testDeployment`** → 點 **執行**
4. 若要求授權 → **檢閱權限** → **進階** → **前往...** → **允許**
5. 日誌應顯示：
   ```
   ✅ index 檔案存在，可以部署
   ✅ Drive 權限正常，資料夾 ID：xxxxx
   ```
6. **部署** → **管理部署作業** → 編輯 → **版本：新版本** → **部署**

#### B. 使用者（或你自己測試網址時）

1. 前往 [Google 帳戶權限](https://myaccount.google.com/permissions)
2. 找到此日記應用程式 → **移除存取權**
3. 重新開啟部署的網頁 URL
4. 完整走完授權流程 → **允許**
5. 重新整理頁面

### 授權說明（會看到什麼）

新版只要求 **drive.file** 權限，意思是：

> 僅能存取此應用程式建立或開啟的 Google 雲端硬碟檔案

不會讀取你雲端硬碟的其他檔案，比完整 drive 權限更安全。
